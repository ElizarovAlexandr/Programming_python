# Лабораторная работа №6

---

## Постановка задачи

Требуется реализовать функцию `get_currencies(currency_codes, url)`, которая выполняет запрос к сервису Центрального банка РФ по адресу:

```text
https://www.cbr-xml-daily.ru/daily_json.js
```

и возвращает информацию о курсах валют, указанных в списке `currency_codes`.

### Основные требования

* результат должен представлять собой словарь;
* ключами словаря являются коды валют (USD, EUR и т.д.);
* значениями словаря являются актуальные курсы валют;
* при возникновении ошибки функция должна возвращать `None`;
* для выполнения запроса необходимо использовать библиотеку `requests`.

### Необходимо обработать следующие ситуации

* отсутствуют данные о валютах в ответе сервера;
* запрошенная валюта отсутствует в полученных данных;
* ошибка выполнения HTTP-запроса.

## Ход выполнения работы

### Первый этап

На начальном этапе была реализована базовая обработка исключений с использованием конструкции `try-except`.

Информация об ошибках выводилась в консоль при помощи стандартных средств языка Python.

### Второй этап

Для исключения повторяющегося кода был создан декоратор `@log`.

Декоратор берет на себя обработку исключительных ситуаций и регистрацию возникающих ошибок.

Благодаря этому основной код функции стал более компактным и читаемым.

### Третий этап

В программу был добавлен модуль `logging`.

Теперь сведения о работе приложения записываются в файл журнала `log.log`.

Для каждой записи фиксируются:

* дата и время события;
* уровень сообщения;
* текст сообщения.

Используются уровни:

* INFO;
* WARNING;
* ERROR.

Для проверки корректности работы использовался модуль `unittest`.

В процессе тестирования были рассмотрены следующие случаи:

Для имитации работы внешнего сервиса использовались возможности библиотеки `unittest.mock`, что позволило выполнять тестирование без обращения к реальному API.
